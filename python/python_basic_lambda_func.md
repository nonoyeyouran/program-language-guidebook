
## lambda 与 def 的对比

| 特性            | lambda 函数            | def 函数            |
|-----------------|-----------------------|---------------------|
| 定义方式        | `lambda x: expression`| `def name(x):`      |
| 命名            | 匿名                  | 有名字             |
| 代码复杂度      | 仅单表达式            | 可多行代码         |
| 返回值          | 自动返回表达式结果    | 需要 `return` 语句 |

## 使用场景

1. **短小的回调函数**: 如 `map()`、`filter()`、`sorted()`。
2. **一次性使用**: 不需要重复调用的小功能。
3. **函数式编程**: 与高阶函数结合使用。

## 注意事项

1. **限制性**: lambda 不能包含复杂的逻辑（如循环或多条件语句）。
2. **可读性**: 过于复杂的 lambda 函数会降低代码可读性，此时建议用 `def`。
3. **调试困难**: 因为没有名字，调试时不如命名函数方便。
